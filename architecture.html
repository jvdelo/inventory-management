Filter Data Flow — End to End
1

User selects a filter

FilterBar.vue updates a ref in the useFilters() singleton — e.g. selectedLocation.value = 'London'. All views share the same refs, so the change is visible everywhere instantly.

2

Watch triggers a reload

Each view has a watch([selectedLocation, selectedCategory, ...], loadData). When any watched ref changes, loadData() fires automatically.

3

API call with query params

getCurrentFilters() builds a params object and api.js serialises it via URLSearchParams. Example request: GET /api/orders?warehouse=London&month=2025-09

4

Backend filters in-memory data

FastAPI route receives query params and calls apply_filters() (warehouse → category → status), then filter_by_month() if a month param is present. All filtering is pure Python iteration — no SQL.

5

Pydantic validates & serialises

Each dict in the filtered list is validated against its Pydantic model (e.g. Order) and serialised to JSON. FastAPI sends it back as a typed HTTP 200 response.

6

Vue re-renders via computed properties

The response is stored in a ref (e.g. orders.value = data). Computed properties that depend on it recalculate (totals, counts, sorted lists), and the template re-renders automatically — no manual DOM work.
